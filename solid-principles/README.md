#### Single Responsibility Principle (SRP)

- Created By: Robert C. Marring (Uncle bob)
- Definition: Each Class/function/Software module should have a single responsibility i.e. one and only one reason to change.
- Responsibility: Persistence, Logging, Validation and Business Logic
- Strive for high cohesion and loose coupling
- Keep classes small, focussed and testable
